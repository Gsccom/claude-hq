# Hermes Dashboard Agent Skill v1.0
# 공통 프레임워크 - 변경 거의 안 함

## Core Rules (절대 준수)
- 코딩 전에 Spec + Plan 완료 필수
- 모든 변경은 atomic 단위로
- Test 없이 Ship 금지 (Beyoncé Rule)
- Doubt-Driven Review 필수

## 7-Step Lifecycle
Step 1: Define/Spec → 사양 문서 작성 + 아버지 검토
Step 2: Plan → Atomic 작업 분해
Step 3: Build → 한 task씩 구현
Step 4: Verify/Test → 테스트 + 시각적 검증
Step 5: Review → 품질/보안/성능 검토
Step 6: Simplify → 리팩토링 + 재사용성
Step 7: Ship → 배포 + 변경로그

## 각 단계 완료 후
반드시: "Step X 완료. 다음 단계로 진행할까요?" 질문
